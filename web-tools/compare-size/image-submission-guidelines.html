Image Submission Guidelines

Last updated:

December 9, 2023



Guideline Summary

  • No content which is sexually explicit, obscene, immoral, or content from movies

  • Keep graphics simple but interesting

  • Make it accurate but easy to understand in silhouette form

  • All angles of the object should be perfectly perpendicular to the side of the object depicted

  • The color of the object must be exactly #555555 and an alpha of 1.0

  • The object should generally only be one, solid object

  • All stroked paths should be converted to filled paths

  • The page size of the SVG should be the same size as the object's bounding box (no white space)



Note: Try to make your graphic match these guidelines as best as possible, however, don't worry if your graphic isn't perfect. Slight modifications may be made to your graphic so that it fulfills the guidelines.


Depicted Content

Anything which is an object can be uploaded

EXCEPT

anything which is sexually explicit, obscene, immoral, or content from movies.



Simplicity

To keep the tool fast and responsive as well as making the visualizations easy to understand, object images should be simple. When designing an object, make it simple but not too simple. Objects with too much or too little detail may be declined. Detail makes the tool more interesting but it also makes the tool slower, so a pleasing middle ground is recommended.




Accuracy

Make sure your object silhouette is as accurate as possible in most areas! Generally speaking, the more accurate, the better. It is recommended that you trace the silhouette based on a reference image. There are some cases where accuracy should be lessened though.


For example, if you're making a graphic of an animal, try to make the legs straight and perpendicular to the ground. This is usually not according to the reference image but it make it easier to understand in a silhouette.



Angles

Side images

should be a view of the left side of the object with the object facing left. The width of the image should be the length of the object. The height of the image should be the height of the object.



Front images

should be a view of the front of the object. The width of the image should be the width of the object. The height of the image should be the height of the object.



Top images

should be a view of the top of the object with the object facing left. The width of the image should be the length of the object. The height of the image should be the width of the object.




Note: All angles should be perfectly perpendicular to the side, front, or top of the object (i.e. no 45 front and side angles, etc.).


Additional Guidelines

The color of the object should be exactly #555555 (RGB: 85, 85, 85) with alpha of 1.0 (#FF). The object should be on a fully transparent background (alpha of 0.0).



The final image should be only one, solid object (in most cases). If you have stroked paths, make sure to convert them to a filled path.



Make sure the page size of the SVG is exactly the same as the size of the object (no white space). This does NOT mean that the image must be a certain meters/pixel etc. This is simply for eliminating white space to avoid errors when the tool sizes the image. Removing white space can be achieved in Inkscape using File > Document Properties > Resize to content.
